Methodology

System of Tagging

The newspaper articles and the two book sources have been marked up using XML tagging. The tagging system highlights a variety of elements and attributes. Each article and chapter of the books were verified against our Relax NG schema to ensure consistent tagging.

div

structural -- The "div" element is always followed by a "type" attribute.

The "type" attribute points to one of the following descriptive labels when associated with "div": "preface" "chapter" "articleReference" "newspaperArticle"

dialogue

This element indicates an exchange between any of the voice tags (mascVoice, nellVoice, femVoice). Dialogue was created for presentation purposes to highlight conversations between people in both the articles and the books' chapters.

unclear

This element was created to indicate the possibility of poor transcription due to an unclear image transcribing from. The transcriber gives the best attempt to determine what was written; however, the "unclear" tag indicates if the is any uncertainty. This is a self-closing tag.

ref

The "to" attribute points to one of the following descriptive labels: "newspaperSeries" "nellNelson" "theTimes"

statistic

The "info" attribute points to one of the following descriptive labels: "vague" or "specific"

industry

With this we tagged any mention of a specific type of industry.

desc

This element indicates a description of something based off the associated "of" attribute.

The "of" attribute points to one of the following descriptive labels: "womensLabor" "workingClass" "middleClass" "upperClass"

nationality

This element contains any references to a specific nationality. The nationality is defined by the associated "nation" attribute.

The "nation" attribute points to one of the following descriptive labels: "America" "Spain" "Africa" "Sweden" "China" "Germany" "Hungary" "Ireland" "undefined"


Return to Top of Page

occupation

This element contains any references to a specific occupation.


Return to Top of Page

num

This element refers to a number. The "type" and "amount" attribute must follow this element.

The "type" attribute points to one of the following descriptive labels when associated with "num": "wage" "cost" "profit" "price"

The "amount" attribute contains a number with decimal value.


Return to Top of Page

person

This element contains reference to a person. The element "person" should be defined by the attribute "who" and/or "gender".

The "who" attribute points to one of the following descriptive labels when associated with "person": "forePerson" "workingGirl" "employer" "employee" "boy"

The "gender" attribute points to one of the following descriptive labels: "male" or "female"


Return to Top of Page

date

This element contains any references to dates. "date" must be followed by one of these attributes: "when" "year" "month" "day".

The "when" attribute defines the date in this format: year-month-day

The "year" attribute defines the four digit representation of a year

The "month" attribute defines the two digit representation of a month

The "day" attribute defines the two digit representation of a day

This element contains quoted material. Each quote should have a defined "speaker" attribute

The "speaker" attribute can indicate one of these descriptive labels: "Tennyson" "factoryInspector" "laborCommissioner" "workGirl"
